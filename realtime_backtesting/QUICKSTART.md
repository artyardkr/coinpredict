# 🚀 빠른 시작 가이드

실시간 비트코인 백테스팅 시뮬레이터를 5분 안에 시작하세요!

## 1️⃣ 설치 (1분)

### 패키지 설치
```bash
cd realtime_backtesting
pip install -r requirements.txt
```

필요한 패키지:
- ✅ streamlit (대시보드)
- ✅ pandas, numpy (데이터 처리)
- ✅ plotly (차트)
- ✅ yfinance (데이터 수집)
- ✅ scikit-learn (모델)

## 2️⃣ 실행 (30초)

### 기본 실행
```bash
streamlit run dashboard.py
```

브라우저가 자동으로 열리며 http://localhost:8501 접속

### 다른 포트로 실행
```bash
streamlit run dashboard.py --server.port 8502
```

## 3️⃣ 사용 (2분)

### Step 1: 전략 선택
사이드바에서 원하는 전략 선택:
- **long_only**: 안전하게 (상승 예측시만 매수)
- **threshold**: 추천! (1% 이상 예측시만 매수)
- **long_short**: 공격적으로 (하락시 공매도)

### Step 2: 파라미터 조정
```
초기 자본: $10,000
Threshold: 1.0%
거래 비용: 0.1%
데이터 기간: 365일
시뮬레이션 속도: 5일/초
```

### Step 3: 시작!
**▶️ 시작** 버튼 클릭 → 실시간 시뮬레이션 관찰

## 4️⃣ 결과 확인 (1분)

시뮬레이션이 끝나면 다음 지표 확인:

### 핵심 지표
- ✅ **총 수익률**: 전체 수익률 (%)
- ✅ **샤프 비율**: 위험 대비 수익 (1.0 이상이면 좋음)
- ✅ **최대 낙폭**: 최악의 손실 구간 (낮을수록 좋음)
- ✅ **승률**: 거래 성공률 (50% 이상이면 양호)

### 차트
- 📈 **가격 차트**: 실제 vs 예측 가격
- 💰 **포트폴리오**: 전략 vs Buy-and-Hold
- 📊 **거래 내역**: 최근 거래 기록

## 💡 추천 설정

### 초보자용 (안전)
```
전략: long_only
초기 자본: $10,000
거래 비용: 0.1%
데이터 기간: 365일
```

### 중급자용 (균형)
```
전략: threshold
Threshold: 1.0%
초기 자본: $10,000
거래 비용: 0.1%
데이터 기간: 365일
```

### 고급자용 (공격적)
```
전략: long_short
초기 자본: $10,000
거래 비용: 0.1%
데이터 기간: 730일
```

## 🎯 전략 비교 실험

### 실험 1: Threshold 최적화
여러 threshold 값을 테스트하여 최적값 찾기:

| Threshold | 예상 결과 |
|-----------|----------|
| 0.5% | 거래 많음, 변동성 높음 |
| 1.0% | 균형 잡힘 (추천) |
| 2.0% | 거래 적음, 안정적 |
| 3.0% | 거래 매우 적음 |

### 실험 2: 전략 비교
3가지 전략 순차 실행 후 비교:

1. **Long-Only** 실행 → 수익률 기록
2. **Threshold 1.0%** 실행 → 수익률 기록
3. **Long-Short** 실행 → 수익률 기록

가장 높은 샤프 비율을 가진 전략이 최적!

### 실험 3: 거래 비용 영향
거래 비용을 변경하여 민감도 확인:

| 거래 비용 | 영향 |
|----------|------|
| 0.05% | 거래 활발 |
| 0.10% | 균형 |
| 0.20% | 거래 제한 |

## 📊 예상 결과

### 일반적인 성과 범위
- **총 수익률**: -20% ~ +100%
- **샤프 비율**: -1.0 ~ +3.0
- **최대 낙폭**: -30% ~ -5%
- **승률**: 40% ~ 60%

### Buy-and-Hold vs 전략
- Buy-and-Hold가 더 좋을 수도 있습니다
- 시장 상황에 따라 전략 성과는 변동
- 여러 기간을 테스트해보세요
